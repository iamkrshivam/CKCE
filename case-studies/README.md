
# Case Studies – Framework

Real‑world cybersecurity incidents provide valuable lessons. This directory defines how contributors may submit **anonymised case study skeletons**. No sensitive data, no actual breach details, only high‑level analysis frameworks.

## Definition
A cybersecurity case study is a structured description of a security incident (breach, malware outbreak, etc.) focusing on:
- What happened (general timeline).
- Root cause (domain‑specific vulnerability).
- Impact (qualitative, no financial numbers unless public).
- Defensive lessons and improvements.

## Structure rules
Use `/templates/CASE_STUDY_TEMPLATE.md`. Each case study must:
- Anonymise all organisation and individual names.
- Be based on public information or permission from the affected party.
- Include a “Mitigations” section that maps to CKCE domains.

## Domain mapping
Each case study must list affected domains (e.g., “Network Security, IAM”).
