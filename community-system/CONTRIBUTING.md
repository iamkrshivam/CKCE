# Contributing to CKCE

Thank you for your interest! All contributions must follow the **framework‑only** rule – no code, no real attacks, no sample write‑ups. You are helping to build the *structure* and *documentation* that will later host real content.

## Workflow
1. Fork the repository.
2. Create a branch: `feature/your-change`.
3. Make changes **only to Markdown files** and folder structure.
4. Follow the templates in `/templates`.
5. Submit a pull request (PR) against the `main` branch.

## What is accepted
- Adding missing sections to domain READMEs.
- Improving governance documents.
- Creating new learning path skeletons.
- Fixing typos or structural inconsistencies.

## What is rejected
- Any code, scripts, or binaries.
- Real vulnerability examples, exploit code, or attack instructions.
- Duplicate content (check existing files).

## Quality standards
- Markdown must be valid and readable.
- No external links without justification (placeholders allowed).
- Defensive language only (e.g., “prevent XSS” not “how to execute XSS”).

See `REVIEW_PROCESS.md` for how PRs are evaluated.
