# Pull Request Review Process

1. **Initial check** (automated) – Verifies that no forbidden file types (.exe, .py, .bin) are present.
2. **Domain maintainer assignment** – Based on the folders changed.
3. **Review criteria**:
   - Compliance with defensive‑only rule.
   - Adherence to folder structure and templates.
   - No duplicate content.
   - Clear, professional English.
4. **Decision**:
   - **Approve** – PR merged.
   - **Request changes** – Contributor updates.
   - **Reject** – Explanation given (e.g., violates ethical policy).
5. **Merge** – Only maintainers can merge after two approvals for structural changes.

All reviews are public and constructive.
