# CTF Hub – Framework

This directory will contain **structured skeletons for Capture The Flag (CTF) write‑ups**. No actual write‑ups, flags, or live challenges are included. Contributors will later add educational, defensive CTF analyses.

## Allowed CTF content
- Description of the challenge concept (e.g., “a vulnerable web app that requires SQL injection to retrieve a flag”).
- Step‑by‑step **defensive** lessons: how the vulnerability works, how to fix it, and how to detect it.
- **No actual flag strings** – use placeholders like `FLAG{example}`.
- **No automated exploit scripts** – only manual reasoning.

## Structure
Each write‑up skeleton must be placed under `/domains/<domain>/labs/` or `/ctf-hub/writeups/<domain>/`. Use `/templates/WRITEUP_TEMPLATE.md`.

## Ethical statement
CTF content must never encourage attacking real systems. All challenges are for isolated, authorised lab environments.
