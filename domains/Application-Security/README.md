# Application Security Domain

What is it? – Securing software applications throughout the SDLC.  
Why important? – Majority of breaches exploit application vulnerabilities.  
Industry usage – SAST/DAST, code reviews, threat modelling.  
Contributions allowed – Secure coding principles, OWASP Top 10 explanations, authentication design patterns.  
Difficulty levels – Beginner (input validation), Intermediate (session management), Advanced (RASP, API security), Research (supply chain security).  
Connections – Cloud Security (serverless), IAM (authZ/authN), Cryptography (hashing).  
Ethical – No actual SQLi or XSS payloads. Only defensive mitigations.
