# Information Security Domain

What is it? – Protection of information assets (confidentiality, integrity, availability).  
Why important? – Covers policies, risk management, and data classification.  
Industry usage – ISO 27001, NIST SP 800‑53, data handling procedures.  
Contributions – Risk assessment frameworks, security awareness topics, policy templates.  
Difficulty – Beginner (CIA triad), Intermediate (risk matrices), Advanced (legal compliance), Research (privacy‑preserving tech).  
Connections – GRC, Data Security, IAM.  
Ethical – No actual PII or trade secrets. Only generic models.
