# Contributing to Network Security Domain

Thank you for your interest in contributing to the Network Security domain. Please follow the global contribution guidelines in `/community-system/CONTRIBUTING.md` plus the rules below.

## Domain‑specific rules
1. **Focus on defence** – Topics like “how to break a firewall” are not allowed; “how to properly configure a firewall rule set” is allowed.
2. **No live packet captures** – PCAPs containing real traffic may contain sensitive data. Only synthetic, anonymised, or described examples are allowed.
3. **Citing sources** – Academic papers, RFCs, and vendor documentation must be properly referenced.
4. **File placement** – Ensure your contribution lands in the correct difficulty subdirectory.
5. **Cross‑domain** – When relevant, link to other domains (e.g., Cloud Security for VPC security groups).

## Review process
A domain maintainer (listed in `/community-system/MAINTAINERS.md`) will review your pull request within 7 days. Expect feedback on structure, correctness, and defensive posture.
