# Network Security Domain

## What is Network Security?
Network Security encompasses the policies, practices, and technologies used to monitor, protect, and defend computer networks from unauthorised access, misuse, modification, or denial of service.

## Why is it important in cybersecurity?
Networks are the backbone of modern organisations. A single vulnerability in network infrastructure can expose entire systems to attackers. Network Security is the first line of defence.

## Real‑world usage in industry
- Firewalls, IDS/IPS, VPNs, network segmentation
- Zero Trust Network Access (ZTNA)
- Secure network design for cloud and hybrid environments

## Types of contributions allowed (theoretical only)
- **Theoretical knowledge** – Definitions, protocols (TCP/IP, DNS, BGP), network architecture patterns.
- **Research notes** – Summaries of academic papers, novel defence mechanisms.
- **Case studies (future)** – High‑level analysis of network breaches (no raw PCAPs, no exploit code).
- **Tools (future)** – Descriptions of defensive tools (e.g., Wireshark usage, Snort rules) – no code.

## Difficulty levels
| Level       | Description |
|-------------|-------------|
| Beginner    | OSI model, subnetting, basic firewall concepts. |
| Intermediate | VLANs, routing protocols, IDS/IPS configuration. |
| Advanced     | Network forensics, BGP hijacking detection, SDN security. |
| Research     | New network attack surfaces (e.g., 5G, QUIC), AI for network defence. |

## Connections to other domains
- **Cloud Security** – Virtual networks, security groups.
- **Incident Response** – Network‑based detection and containment.
- **Cryptography** – IPsec, TLS, SSH.
- **Threat Intelligence** – Network IoCs, malicious traffic patterns.

- Use the `/templates/DOMAIN_TEMPLATE.md` for new topics.
