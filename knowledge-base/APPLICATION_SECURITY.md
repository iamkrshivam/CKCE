# Application Security – Scope

## Core topics
- Secure Software Development Lifecycle (SDLC)
- Threat modelling (STRIDE, DREAD, PASTA)
- Common vulnerability classes (OWASP Top 10, CWE)
- Static Application Security Testing (SAST)
- Dynamic Application Security Testing (DAST)
- Interactive Application Security Testing (IAST)
- Software Composition Analysis (SCA)
- API security (REST, GraphQL, authentication, rate limiting)
- Web application firewalls (WAF)
- Secure coding guidelines (input validation, output encoding)

## Relationship with other domains
- **Cryptography** – Hashing, encryption, key management in apps.
- **IAM** – Authentication (OAuth2, OpenID Connect) and authorisation.
- **Cloud Security** – Serverless security, containerised apps.
- **Vulnerability Management** – Prioritising application flaws.

## Learning structure
- **Beginner** – OWASP Top 10 concepts, secure coding basics, input validation.
- **Intermediate** – SAST/DAST tool usage (theoretical), threat modelling exercises.
- **Advanced** – API security, RASP, software supply chain security.
- **Research** – New vulnerability classes, AI‑assisted code review.

Contributors must not include live exploit code or step‑by‑step attack instructions. Only defensive mitigations and secure design patterns.
