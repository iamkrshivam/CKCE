# AI Security Learning Path

**Target career**: AI Security Researcher, Machine Learning Security Engineer  
**Prerequisites**: Basic programming concepts, mathematics (linear algebra, statistics) – theoretical understanding.  
**Duration**: 12–15 months (research‑oriented).

## Phase 1: Foundations of AI/ML
- **Domains**: Information Security (Beginner), Cryptography (Intermediate – for privacy concepts)
- **Skills**: ML lifecycle (data, training, inference), model types (classification, generation).
- **Prerequisites**: None, but numerical literacy required.

## Phase 2: Adversarial Machine Learning (Defensive)
- **Domains**: Application Security (Advanced), Research (Beginner)
- **Skills**: Adversarial examples (theory), evasion attacks, defensive distillation.
- **Prerequisites**: Phase 1.

## Phase 3: AI for Cyber Defence
- **Domains**: SOC (Advanced), Threat Intelligence (Intermediate)
- **Skills**: Using ML for anomaly detection, malware classification, threat hunting.
- **Prerequisites**: Phase 2.

## Phase 4: Privacy and Ethical AI
- **Domains**: Data Security (Advanced), GRC (Intermediate)
- **Skills**: Differential privacy, federated learning, AI compliance (EU AI Act).
- **Prerequisites**: Phase 3.

## Capstone Project (skeleton)
- **Project**: Design a defensive ML model for network intrusion detection – description of features and evaluation metrics.
- **Domains**: Network Security, SOC, Research.
- **Outcome**: Research proposal or model architecture outline (no code).
