# Cloud Security Learning Path

**Target career**: Cloud Security Engineer, DevSecOps Engineer  
**Prerequisites**: Basic networking, operating systems, and familiarity with at least one cloud provider (conceptual).  
**Duration**: 6–8 months.

## Phase 1: Cloud Fundamentals & Shared Responsibility
- **Domains**: Cloud Security (Beginner), Network Security (Beginner)
- **Skills**: Cloud service models (IaaS/PaaS/SaaS), shared responsibility, VPC basics.
- **Prerequisites**: None.

## Phase 2: Identity and Access Management in Cloud
- **Domains**: IAM (Intermediate), Cloud Security (Intermediate)
- **Skills**: IAM roles/policies, least privilege, federation concepts.
- **Prerequisites**: Phase 1.

## Phase 3: Data Protection and Compliance
- **Domains**: Data Security (Intermediate), GRC (Beginner)
- **Skills**: Encryption in cloud, key management, compliance frameworks (CIS, SOC2).
- **Prerequisites**: Phase 2.

## Phase 4: Container and Serverless Security
- **Domains**: Application Security (Advanced), Cloud Security (Advanced)
- **Skills**: Docker/Kubernetes security primitives, serverless risk model.
- **Prerequisites**: Phase 3.

## Capstone Project (skeleton)
- **Project**: Design a secure multi‑account cloud architecture (AWS/Azure/GCP) – diagram and policy outline.
- **Domains**: Cloud Security, IAM, Network Security.
- **Outcome**: Architecture diagram with security controls annotated.
