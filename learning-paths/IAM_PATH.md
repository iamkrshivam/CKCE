# Identity and Access Management (IAM) Learning Path

**Target career**: IAM Engineer, Identity Architect  
**Prerequisites**: Basic networking, understanding of directories (conceptual).  
**Duration**: 5–7 months.

## Phase 1: IAM Core Concepts
- **Domains**: IAM (Beginner), Cryptography (Beginner)
- **Skills**: Authentication, authorisation, accounting (AAA), directory services (LDAP).
- **Prerequisites**: None.

## Phase 2: Modern Authentication Protocols
- **Domains**: IAM (Intermediate), Application Security (Intermediate)
- **Skills**: OAuth2, OpenID Connect, SAML 2.0, Kerberos (theory).
- **Prerequisites**: Phase 1.

## Phase 3: Privileged Access Management (PAM)
- **Domains**: IAM (Advanced), Endpoint Security (Intermediate)
- **Skills**: Just‑in‑time access, session recording, password vaulting.
- **Prerequisites**: Phase 2.

## Phase 4: Identity Governance and Cloud IAM
- **Domains**: IAM (Advanced), Cloud Security (Intermediate)
- **Skills**: Access certifications, role mining, cloud IAM (AWS IAM, Azure AD).
- **Prerequisites**: Phase 3.

## Capstone Project (skeleton)
- **Project**: Design an IAM architecture for a 500‑employee company (on‑prem + cloud).
- **Domains**: IAM, Cloud Security.
- **Outcome**: High‑level design document with authentication flows and role definitions.
