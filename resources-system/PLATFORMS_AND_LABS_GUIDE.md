# Platforms and Labs Guide – Classification Schema

This guide covers hands‑on cybersecurity lab platforms (cloud sandboxes, VM libraries, practice ranges).

## Platform categories
- **Cloud lab environments** – AWS security labs, Azure sandboxes.
- **Local VM images** – VulnHub, Metasploitable (defensive analysis only).
- **CTF platforms** – picoCTF, CTFd instances (educational, not real‑world).
- **Simulated SOC platforms** – Splunk workshops, Elastic Security labs.

## Metadata required
```markdown
- Platform name: [Name]
- Focus domains: [list]
- Difficulty: Beginner / Intermediate / Advanced
- Setup required: (Browser‑only / Local VM / Cloud account)
- Cost: Free / Paid
- Defensive orientation: (Detection / Analysis / Hardening)
Ethical constraint
Platforms that host “live” vulnerable applications must be used only in isolated, authorised environments. Contributors must include a warning: “Do not attack systems you do not own or have written permission to test.”
