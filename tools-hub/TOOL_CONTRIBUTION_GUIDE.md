# Tool Contribution Guide

1. **Verify the tool is defensive** – If the tool can be used offensively, you must explain only its defensive applications and explicitly warn against misuse.
2. **No download links** – Instead, say “available from the official website” or “included in most Linux distributions”.
3. **No configuration snippets that weaken security** – e.g., disabling TLS.
4. **Respect licensing** – Mention the tool’s license (open source preferred).
5. **Review process** – A maintainer will check for compliance with the defensive‑only rule.
